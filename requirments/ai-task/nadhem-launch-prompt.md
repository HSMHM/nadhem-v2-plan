# البرومبت — الصقه في شات Claude Code

---

أنت مسؤول عن بناء صفحة ويب React.js تعرض **خطة تطوير منتج "نظم" (Nadhem)** لعام 2026.

"نظم" منتج قائم فعلياً — نظام إدارة فعاليات يخدم عملاء حاليين. المطلوب عرض الوضع الحالي للمنتج وخطة تطويره بشكل جميل ومفهوم.

## الملفات المرجعية

اقرأ هذه الملفات بالترتيب قبل أي كود:

1. **`requirements/ai-task/nadhem-v2-tasks.md`** — ملف المهام الرئيسي (يخبرك بكل شيء: الأقسام، التصميم، الرسوم، البيانات)
2. **`requirements/roadmap-file/خارطة الطريق - منتج نظم.pdf`** — أهم ملف: الأرقام الحقيقية (نسب الاعتماد، الرضا، المنافسين، التحديات، التطويرات)
3. **`requirements/current-product/current01.pdf`** — وصف المنتج وخصائصه
4. **`requirements/current-product/current02.pdf`** — عرض المزايا والتكاملات
5. **`requirements/current-product/current03.pdf`** — مزايا إضافية

ملفات txt هي نسخ نصية من الـ PDFs — استخدمها إذا صعبت قراءة الـ PDF.

## قواعد حرجة

- **لا Emoji نهائياً** — استخدم Font Awesome Jelly فقط (`fa-thin fa-...`)
- **Font Awesome Kit:** `<script src="https://kit.fontawesome.com/d64cd9d612.js" crossorigin="anonymous"></script>` في index.html
- **لا تثبّت lucide-react** — احذفها إن وجدت
- **الشعار الأبيض** (`white-logo.png`) في القائمة الجانبية والصفحة الداكنة
- **الشعار الملون** (`colored-logo.png`) في باقي الأماكن
- **كل النصوص بالعربية — RTL**
- **كل البيانات Hardcoded — لا قواعد بيانات ولا API**
- **بدون مصطلحات تقنية معقدة** — الخطة لاجتماع وليست لمطورين
- **كل ملف أقل من 300 سطر**
- **اختبر بعد كل 3-4 ملفات**

## الشعارات

```
requirements/Logos/white-logo.png    ← للقائمة الجانبية (خلفية داكنة)
requirements/Logos/colored-logo.png  ← للاستخدام العام
```

اقرأ `nadhem-v2-tasks.md` بالكامل ثم اقرأ خارطة الطريق PDF. بعد القراءة أخبرني عن خطتك قبل البدء بالكود.
