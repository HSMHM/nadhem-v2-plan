# مهمة: تصحيح إحصائيات ومحتوى خطة التسويق

## المشكلة في الأرقام
بطاقة "منظمة غير ربحية" تعرض الرقم 6 بدلاً من 6,300. السبب أن القيمة مكتوبة كنص `'6,300+'` وليس كرقم. يجب تحويل كل القيم إلى أرقام صحيحة مع suffix منفصل.

---

## التعديلات المطلوبة

### 1. تصحيح بطاقات الإحصائيات (`mktStats`)

```js
export const mktStats = [
  { icon: 'building-ngo', value: 6300, suffix: '+', label: 'منظمة غير ربحية في المملكة', color: '#2A848A' },
  { icon: 'landmark', value: 200, suffix: '+', label: 'جهة حكومية تنظم فعاليات', color: '#452059' },
  { icon: 'briefcase', value: 150, suffix: '+', label: 'شركة تنظيم فعاليات', color: '#BA5A31' },
  { icon: 'file-contract', value: 7, label: 'عميل نشط حالياً', color: '#10B981' },
  { icon: 'user-xmark', value: 71, label: 'عميل سابق لم يُجدد', color: '#EF4444' },
  { icon: 'bullseye-arrow', value: 25, suffix: '-32', label: 'عميل مستهدف نهاية 2026', color: '#10B981' },
];
```

**تأكد أن مكوّن StatCard يعرض `suffix` بشكل صحيح بعد الرقم.**

### 3. تصحيح بطاقتي الهدف (اكتساب + استعادة)

```jsx
{/* بطاقة الاكتساب */}
<div>10 – 15 عميل جديد</div>
<p>من جميع القطاعات (غير ربحي + حكومي + خاص)</p>

{/* بطاقة الاستعادة */}
<div>8 – 10 عميل مُستعاد</div>
<p>من الـ 71 عميل سابق</p>
```

### 4. تعديل شرائح العملاء (`personas`) — إلغاء محفزات الشراء

من كل شريحة (persona)، احذف الحقل `buyingTrigger` بالكامل. أبقِ فقط: `name`, `icon`, `desc`, `size`, `painPoints`, `priority`.

```js
// احذف من كل شريحة:
// buyingTrigger: '...',
```

وفي المكوّن `PersonasSection.jsx` — احذف أي عرض لـ `buyingTrigger` أو "محفز الشراء".

### 5. إلغاء برنامج الإحالات بالكامل

**أ)** في `channels` — احذف:
```js
// احذف هذا العنصر من القنوات:
{ name: 'العملاء الحاليون (7 عملاء)', ..., actionable: 'برنامج إحالات رسمي مع حوافز' },
```

**ب)** في `gaps` — احذف:
```js
// احذف:
{ gap: 'لا يوجد برنامج إحالات', ... },
```

**ج)** في `mktTasks.productManager` — احذف:
```js
// احذف:
{ id: 'PM.6', task: 'تصميم برنامج الإحالات — الآلية والحوافز والشروط', ... },
```

**د)** في الجدول الزمني (`mktTimeline`) — احذف أي ذكر لـ "إطلاق برنامج الإحالات".

### 6. تعديل LinkedIn — 10 بدلاً من 50

في `channels`:
```js
// القديم:
actionable: 'حملة تواصل مباشر مع 50+ مسؤول شهرياً'
// الجديد:
actionable: 'حملة تواصل مباشر مع 10 مسؤولين شهرياً'
```

في `mktTasks.marketingTeam`:
```js
// القديم:
{ id: 'MKT.4', task: 'إدارة حملة LinkedIn — نشر محتوى + تواصل مباشر مع 50 مسؤول فعاليات شهرياً', ... },
// الجديد:
{ id: 'MKT.4', task: 'إدارة حملة LinkedIn — نشر محتوى + تواصل مباشر مع 10 مسؤولين شهرياً', frequency: 'أسبوعي', duration: '1 ساعة', priority: 'high' },
```

### 7. "غرف التجارة" → "الغرف التجارية"

في `channels` ابحث عن "غرف التجارة" واستبدلها بـ "الغرف التجارية" في كل مكان.

### 8. إلغاء مهمة التسجيل في اعتماد (مسجلون فعلاً)

في `mktTasks.productManager` — احذف:

## ملخص التناقضات التي يجب مراجعتها

بعد تطبيق التعديلات أعلاه، ابحث في **كامل المشروع** عن هذه القيم وتأكد من تحديثها في كل مكان:

| ابحث عن | استبدل بـ | الملفات المحتملة |
|---------|----------|-----------------|
| `30-50` (leads شهرياً) | `5-8` | marketing data + tasks + dashboard |
| `20-30` (عملاء جدد) | `10-15` | marketing data + targets + timeline |
| `11-14` (مستعادون) | `8-10` | marketing data + winback + targets |
| `35-50` (إجمالي مستهدف) | `25-32` | marketing data + stats + timeline |
| `8-12` (demos شهرياً) | `3-5` | marketing tasks + dashboard |
| `50` (LinkedIn شهرياً) | `10` | marketing channels + tasks |
| `200+` (قائمة مستهدفة) | `100` | marketing tasks |
| `برنامج إحالات` أو `الإحالات` | **احذف** | channels + gaps + tasks + timeline |
| `محفز الشراء` أو `buyingTrigger` | **احذف** | personas data + PersonasSection |
| `التسجيل في منصة اعتماد` | `تفعيل التواجد في اعتماد` | tasks + channels + timeline |
| `لا يوجد نظام CRM` | `تفعيل نظام CRM` | gaps |
| `لا يوجد عرض تجريبي ذاتي` | **احذف** | gaps + tasks + timeline |
| `لا يوجد برنامج إحالات` | **احذف** | gaps |
| `غرف التجارة` | `الغرف التجارية` | channels + tasks |
| `'6,300+'` (كنص) | `6300` (كرقم) + `suffix: '+'` | mktStats |

**تنبيه:** بعد كل تعديل، افتح الصفحة وتأكد أن الأرقام المعروضة في البطاقات والرسوم البيانية والجداول متسقة ولا يوجد تناقض بين قسم وآخر.
